Problem: Need interactive presentation framework for live coding demos

Solution:

 - Implemented core DSL for defining presentations:
   - Presentation, Slide, Command, Elaboration types (src/Demo/Core/Types.hs)
   - Monadic builder combinators for slide construction (src/Demo/Core/DSL.hs)
   - Variable substitution with $vN syntax for GHCI->System interop (src/Demo/Core/Variable.hs)

 - Implemented IPC layer for multi-process coordination:
   - Length-prefixed JSON protocol with message types (src/Demo/IPC/Protocol.hs)
   - Unix socket server for slides process (src/Demo/IPC/Server.hs)
   - Unix socket client for notes/elaboration (src/Demo/IPC/Client.hs)

 - Implemented interpreters for command execution:
   - System command runner with streaming output (src/Demo/Interpreter/System.hs)
   - GHCI evaluator using hint library (src/Demo/Interpreter/Ghci.hs)
   - Presentation loader via hint (src/Demo/Loader.hs)

 - Implemented brick TUI for all three views:
   - Slides view with command display and GHCI mode (src/Demo/UI/Slides.hs)
   - Notes view with IPC sync and recall mode (src/Demo/UI/Notes.hs)
   - Elaboration view for code fragment display (src/Demo/UI/Elaboration.hs)

 - Created four executables:
   - demo: tmux session orchestrator (app/Demo.hs)
   - slides: main presenter interface (app/Slides.hs)
   - notes: synchronized notes display (app/Notes.hs)
   - elaboration: code fragment viewer (app/Elaboration.hs)

 - Implemented comprehensive test suite with 49 passing tests:
   - Variable substitution unit tests (test/Demo/Core/VariableSpec.hs)
   - IPC protocol roundtrip tests (test/Demo/IPC/ProtocolSpec.hs)
   - System command execution tests (test/Demo/Interpreter/SystemSpec.hs)
   - IPC integration tests (test/Integration/IPCSpec.hs)
